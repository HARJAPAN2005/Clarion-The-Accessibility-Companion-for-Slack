# Clarion — 3-minute demo video script

**Goal:** show a working project (judging requires footage of it running), tell an
impact story, and prove all three technologies. Aim for **2:45–3:00**. Record your
screen in the sandbox; talk over it. Prep the props below first.

### Before you hit record
1. `slack run` is live and Clarion responds.
2. Set up one channel, e.g. `#project-atlas`, with:
   - a jargon-heavy message you'll simplify (see prop A),
   - a ~10-message thread you'll catch up on,
   - one image posted (for alt-text).
3. Have the App Home open in a second tab.
4. Optional: set `OPENROUTER_API_KEY` for the best-looking output.

**Prop A (paste this as a message to simplify):**
> "Per my last, let's leverage our bandwidth to circle back and operationalize the Q3 synergies. We need to move the needle on the P1 before EOD — ping me ASAP if there's a blocker."

---

### 0:00–0:20 — The problem (hook)
> *On camera or voiceover, Slack channel visible.*
"This is a normal Slack message. Now read it the way a new hire, a non-native
English speaker, or someone with dyslexia reads it." *(Prop A on screen.)*
"One in four adults has a disability. Slack is where work happens — and it's where
a lot of people quietly get left behind. Meet Clarion: the accessibility layer for
Slack."

### 0:20–0:55 — Simplify (Slack AI capability ①)
> Open the `···` menu on Prop A → **Simplify this message**.
"One click." *(Clarion DMs back a plain-language version — short sentences, jargon
gone, but the P1 and the EOD deadline are still there.)*
"Notice it didn't dumb it down — every decision, owner, and deadline survives.
That's the whole point: clarity without losing meaning."

### 0:55–1:30 — Catch me up (Slack MCP Server ②)
> Go to the long thread → **Catch me up** shortcut (or `@Clarion catch me up`).
"Now a 10-message thread. If you were on leave, catching up is exhausting."
*(Digest appears: The decision / Who owns what / Deadlines / Still open.)*
"Clarion reads the thread through the **Slack MCP Server** — the governed way — and
gives you a structured, plain-language digest. Perfect for anyone with attention
differences, or anyone who was just away."

### 1:30–2:05 — Define a term (Real-Time Search API ③)
> In the channel: `@Clarion what does OKR mean here?`
"New people drown in acronyms. Ask Clarion." *(It answers with how THIS workspace
uses the term first, then the general meaning.)*
"That definition comes from the **Real-Time Search API** — it's grounded in your
live workspace, not a stale wiki. Insider shorthand stops being a barrier."

### 2:05–2:30 — Alt-text + inclusive check
> **Describe image** shortcut on the posted image.
"Screen-reader users hit images with no description constantly. Clarion writes
alt-text in one click." *(Show the alt-text + full description.)*
> Then DM: *"Is this clear? 'Grab the low-hanging fruit and ping me ASAP.'"*
"And it can check a draft before you send it." *(Show the flagged idioms + fixes.)*

### 2:30–2:50 — App Home + close
> Open App Home.
"Everyone controls their own experience — reading level, translation language,
auto alt-text. Clarion meets each person where they are." *(Pan the Home tab.)*
"Clarion uses all three challenge technologies — Slack AI, the MCP Server, and
Real-Time Search — to do one thing: make sure no one gets left out of the
conversation. That's Clarion."

### 2:50–3:00 — Card
> End card: **Clarion — the accessibility layer for Slack.** Small line:
> *Slack Agent Builder Challenge · Slack Agent for Good.*

---

### Recording tips
- Turn on large text / zoom in Slack so the UI is readable in the video.
- Do each action once, cleanly; you can trim pauses in editing.
- If a live model call is slow, the "thinking" status makes the wait look intentional — leave it in.
- Keep the offline fallback as a safety net: if the network drops mid-record, the tools still respond.
- Export 1080p, upload unlisted to YouTube, and paste the link into Devpost.
