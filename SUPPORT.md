# Getting Support

There are several ways to get help with Clarion.

---

## Documentation

Start with the documentation — your question may already be answered:

- [README](README.md) — installation, quickstart, feature overview
- [Developer Guide](docs/DeveloperGuide.md) — local setup, project structure
- [Deployment Guide](docs/Deployment.md) — Socket Mode, HTTP+OAuth, Docker
- [Troubleshooting](docs/Troubleshooting.md) — common issues and fixes
- [FAQ](docs/FAQ.md) — frequently asked questions
- [API Reference](docs/API.md) — internal tool and agent interface documentation

---

## Community Discussions

For questions, ideas, and general conversation, use
[GitHub Discussions](https://github.com/HARJAPAN2005/Clarion-The-Accessibility-Companion-for-Slack/discussions):

- **Q&A** — ask a question and get answers from the community
- **Ideas** — propose new features or improvements
- **Show and Tell** — share how you are using Clarion

---

## Bug Reports

If you have found a bug, please open a
[Bug Report](https://github.com/HARJAPAN2005/Clarion-The-Accessibility-Companion-for-Slack/issues/new?template=bug_report.yml).

Search the [existing issues](https://github.com/HARJAPAN2005/Clarion-The-Accessibility-Companion-for-Slack/issues)
first — your issue may have already been reported.

---

## Accessibility Issues

Found a problem that specifically affects accessibility? Use the dedicated
[Accessibility Issue](https://github.com/HARJAPAN2005/Clarion-The-Accessibility-Companion-for-Slack/issues/new?template=accessibility_issue.yml)
template. These are treated as first-class bugs.

---

## Security Vulnerabilities

**Do not open a public issue for security vulnerabilities.**

See [SECURITY.md](SECURITY.md) for the responsible disclosure process.
